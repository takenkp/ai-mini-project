주어진 정보를 바탕으로 AI 서비스를 심층 분석하고, 시스템 프롬프트에서 요청한 모든 항목을 포함하여 JSON 형식으로 결과를 반환해주십시오.

## 분석 대상 정보:
1.  **서비스 URL**: `{service_url}`
2.  **관련 문서 경로 목록** (분석에 주요 참고 자료):
{document_paths}
3.  **관련 문서 컨텍스트 (RAG 결과)**:
    아래는 서비스의 각 주요 정보 항목(예: 서비스 설명, 핵심 기능 등)에 대해 관련 문서를 검색한 결과입니다.
    이 컨텍스트들을 종합적으로 활용하여 각 항목에 대한 답변을 구성하고, 시스템 프롬프트에서 요청한 JSON 형식으로 전체 결과를 정리해주십시오.
    만약 특정 항목에 대한 컨텍스트가 없거나 부족하더라도, 다른 정보와 URL을 참고하여 최대한 답변을 시도해주십시오.
```text
{rag_context}
```

## 요청 사항:
위 "분석 대상 정보"를 바탕으로 다음 항목들을 추출하여 JSON 형식으로 응답해주십시오.
-   `service_name`: 서비스의 공식 명칭
-   `description`: 서비스의 목적, 주요 가치, 제공 기능 등을 요약한 상세 설명
-   `core_features`: 서비스의 핵심 기능 목록 (각 기능에 대한 간략한 설명 포함 가능)
-   `target_users`: 서비스를 주로 사용하는 대상 사용자 그룹 또는 페르소나
-   `collected_data_types`: 서비스 운영 및 기능 제공을 위해 사용자로부터 수집하는 데이터 유형 목록
-   `service_url_status`: 제공된 서비스 URL의 현재 접속 가능 상태
-   `key_information_source`: 각 주요 정보(이름, 설명, 기능 등)를 어떤 자료(URL 또는 특정 문서명)에서 주로 참고했는지 명시

**출력 JSON 형식 예시 (반드시 이 구조를 따라야 합니다):**
```json
{{
  "service_name": "서비스 이름",
  "description": "서비스에 대한 상세 설명...",
  "core_features": ["기능 1 설명...", "기능 2 설명..."],
  "target_users": ["대상 사용자 그룹 1", "대상 사용자 그룹 2"],
  "collected_data_types": ["수집 데이터 유형 1", "수집 데이터 유형 2"],
  "service_url_status": "접속 가능/불가 등",
  "key_information_source": "주요 정보 출처 (예: 문서명 또는 URL)"
}}
```
만약 특정 정보를 찾을 수 없다면, 해당 필드 값으로 "정보 없음" 또는 "추출 불가"를 사용해주십시오.
